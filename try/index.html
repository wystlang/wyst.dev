UART MMIO write

Source

#module demo.uart

UARTDR :: @mmio u32 = 0x0900_0000
UARTFR :: @mmio u32 = 0x0900_0018
TXFF :: u32 = 1 << 5

pub uart_write :: (byte : u32) {
    while u32@[UARTFR] & TXFF != 0 {
        %nop()
    }
    u32@[UARTDR] = byte
}

Explain facts

  • Source records MMIO intent at both address declarations.
  • Loop condition keeps the explicit %nop() wait body.
  • checked by website tests with wync check and wync build.

AArch64

.Lwait:
  ldr w2, [x1, #0x18]
  tbnz w2, #5, .Lwait
  str w0, [x1]
  ret

Bytes / provenance

  • Profile: qemu-virt-aarch64-el2.
  • Bytes are shown as an illustrative excerpt, not a live compiler result.
  • Provenance: source pattern is checked by website tests; assembly shape is precomputed.

sum_to loop

Source

#module demo.math

pub sum_to :: (n : i32) -> i32 {
    acc : i32 = 0
    i : i32 = 0
    while i < n {
        acc += i
        i += 1
    }
    return acc
}

Explain facts

  • Runtime loop bound remains a counted branch, not an unrolled repeat.
  • Leaf function shape: no frame and no spills in the compact example.
  • checked by website tests against the homepage source and normalized assembly.

AArch64

sum_to:
  mov   w1, #0
  mov   w2, #0
.Lloop:
  cmp   w2, w0
  b.ge  .Ldone
  add   w1, w1, w2
  add   w2, w2, #1
  b     .Lloop
.Ldone:
  mov   w0, w1
  ret

Bytes / provenance

  • Lowering report shape: wync explain lowering.
  • Byte and instruction facts are precomputed examples for reading.
  • No arbitrary code execution is available on this page.

atomic ordering

Source

#module demo.atomic

pub publish :: (flag : @u32, v : u32) {
    #release u32@[flag] = v
}

pub observe :: (flag : @u32) -> u32 {
    return #acquire u32@[flag]
}

Explain facts

  • Ordering is written at the access site.
  • Effects report records acquire/release behavior explicitly.
  • checked by website tests as complete source.

AArch64

publish:
  stlr w1, [x0]
  ret

observe:
  ldar w0, [x0]
  ret

Bytes / provenance

  • Instruction shape is an illustrative excerpt for the ordering pattern.
  • Provenance remains static until a safe WASM or sandboxed compiler path exists.
  • QEMU execution is intentionally out of scope for this stage.
