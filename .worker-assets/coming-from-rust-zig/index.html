Tradeoff guide

Coming from Rust or Zig

Wyst is closer to a semantic assembler than a general systems language. It trades ecosystem, portability, and broad safety machinery for direct ARM64 lowering and deterministic output.

Safety: not memory-safe Backend: no LLVM backend Status: pre-1.0
Familiar concept Wyst equivalent What you gain What you give up
Rust ownership and borrowing Explicit memory access and effects, but no borrow checker. Lower-level code is direct and visible in the source. No memory-safety guarantee from the type system.
Zig-style explicitness Explicit widths, layouts, target facts, and reportable effects. Compiler behavior is deliberately narrow and inspectable. No comparable standard library, cross-platform story, or package ecosystem.
Generics and comptime Explicit generics and source-level requirements in the v0.8-draft surface. Constraints are meant to be visible and reportable. Not a replacement for Rust traits or Zig's broad comptime model.
Mature backends A direct Wyst AArch64 lowering path. No LLVM backend means less hidden optimizer behavior in the emit path. You give up LLVM maturity, target breadth, and optimization depth.
sum_to comparison

The same loop from Rust.

The Rust version can make overflow explicit with wrapping_add, while Wyst is narrower: it is not a Rust replacement, but a small ARM64 language for code where lowering is part of the review surface.

Rust

pub fn sum_to(n: i32) -> i32 {
    let mut acc = 0i32;
    let mut i = 0i32;
    while i < n {
        acc = acc.wrapping_add(i);
        i = i.wrapping_add(1);
    }
    acc
}

comparison snippet; wrapping_add makes the overflow policy explicit

Wyst

#module demo.math

pub sum_to :: (n : i32) -> i32 {
    acc : i32 = 0
    i : i32 = 0
    while i < n {
        acc += i
        i += 1
    }
    return acc
}

Wyst comparison source; smaller surface, no borrow checker, no Rust ecosystem claim

Best fit

Compiler, kernel, and bare-metal experiments where you want to inspect how source maps to AArch64 output.

Main caveat

Wyst is not memory-safe, has no borrow checker, and should not be read as a Rust replacement.

Current limit

ARM64 only, no LLVM backend, Rust bootstrap compiler, and no stable 1.0 compatibility promise yet.
