Wyst Clause-To-Test Conformance Index

Traceability from semantic authority rows to conformance evidence.

This index links every current machine-readable semantic clause in semantic-db.json to the conformance evidence that protects it. The index is intentionally strict about missing evidence: every applicable coverage class is either linked to an existing test or snapshot, marked untested: with a reason, or marked not-applicable: with a reason.

Coverage classes:

  • Positive Source: accepted source or frontend behavior for supported clauses.
  • Negative Diagnostic: rejection of invalid, future, reserved, removed, or unsupported syntax and modes.
  • Execution: runnable or QEMU-backed behavior.
  • IR Snapshot: typed IR, verifier, or IR rendering evidence.
  • Lowering/Disassembly Snapshot: backend lowering or disassembly evidence.
  • ELF/Object/Relocation Snapshot: binary artifact, object, symbol, relocation, or DWARF evidence.
  • Explain-Report Snapshot: wync explain or related report evidence.
  • Reproducibility: build-twice, byte-identical, stable-report, or release evidence.

untested: entries are not completion credit for later semantic work. They are visible debt. A later checklist item that changes, implements, reserves, removes, or promotes a semantic clause must add or update this row before that item can be complete.
