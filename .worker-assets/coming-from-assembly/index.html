Tradeoff guide

Coming from assembly

Wyst keeps AArch64 close, but gives you typed source, structured control flow, modules, layouts, and explain reports when raw assembly becomes hard to audit.

ISA: ARM64 only Output: AArch64 ELF Focus: readable machine intent
Familiar concept Wyst equivalent What you gain What you give up
Labels and branches Named functions, while, if, and explicit branch-like lowering reports. Control flow is reviewable before you read the emitted instruction sequence. Less local freedom to shape every label and fallthrough by hand.
Registers and ABI Typed parameters, native calling convention facts, and register allocation output. ABI and storage facts are surfaced without reverse-engineering the final text. You rely on Wyst's current allocator rather than placing every value yourself.
Load/store sequences Typed memory accesses and MMIO intent annotations. The source says which width and effect you intended. Some instruction selection is mediated by the compiler lowering contract.
Object layout Layout files, regions, entry points, and deterministic ELF emission. Memory map facts are versioned source inputs. No general-purpose assembler ecosystem or macro library compatibility.

Best fit

AArch64 routines where the assembly shape matters, but source-level names, widths, and reports make review easier.

Main caveat

Wyst is ARM64 only and does not try to become a portable assembly abstraction.

Current limit

It is a young compiler, not a replacement for mature assemblers, linkers, or debug toolchains.
