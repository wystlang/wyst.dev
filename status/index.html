Project status

Current implementation status.

Wyst is an early ARM64 bare-metal and kernel-oriented language and assembler. This page separates what works today from draft design, experimental surfaces, and non-goals.

Version: v0.8-draft Target: ARM64 only Compiler: Rust bootstrap compiler Evidence: QEMU-tested fixtures

Implemented today

  • AArch64 ELF output for bare-metal projects.
  • wync check, wync build, wync fmt, and project builds.
  • Lowering, effect, and storage explain reports with explicit provenance fields.
  • QEMU fixtures and deterministic rebuild proof in the release evidence path.
  • LSP and editor-oriented diagnostics.

Specified but not implemented

  • Complete v0.8-draft conformance coverage across all documented corners.
  • Broader report freshness checks for every site example that looks generated.
  • Editable playground support for check, fmt, and explain.

Experimental

  • Source requirements and effect-deny workflows.
  • MMIO intent annotations and report display.
  • Storage contract reporting for buffers, arenas, and handles.
  • The public v0.8-draft language surface before a 1.0 compatibility promise.

Not planned

  • No LLVM backend; lowering is meant to be inspectable and part of the contract.
  • No claim that Wyst is memory-safe; invalid memory can still fault or misbehave.
  • No transparent multi-ISA portability layer in the current project.
  • No optimizer that relies on compiler-exploitable undefined behavior.

Release history

v0.8-draft
Current draft tracked by the website: source requirements, MMIO intent, report provenance, and release-evidence language.
pre-1.0
No compatibility promise yet. Syntax, report schemas, and project layout can still change while the compiler hardens.

Current limits

  • Target: ARM64 only
  • Evidence: QEMU-tested
  • Compiler: Rust bootstrap compiler
  • Status: not self-hosting
  • Safety: not memory-safe
  • Backend: no LLVM backend
  • Version: v0.8-draft
